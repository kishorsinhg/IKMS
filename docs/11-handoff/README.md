# Handoff Convention

Use `ai-handoff.md` as the single current-state handoff for future Codex sessions.

## Rules

- Keep exactly one active handoff file: `docs/11-handoff/ai-handoff.md`.
- Update it whenever the checkpoint, blocker, next task, or implementation direction changes.
- Keep it short and current; replace stale details instead of appending a long history.
- Record durable decisions in `docs/12-decisions/decision-log.md`, not only in the handoff.
- Record execution history in `docs/13-worklog/`, not in the handoff.

## Minimum Content

- Current goal or active feature
- Relevant artifact paths
- Current implementation checkpoint
- Constraints or blockers
- Next recommended task
