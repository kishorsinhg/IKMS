# Worklog Convention

Use daily worklog files instead of one long shared log.

## File Naming

- Create one file per date using `YYYY-MM-DD.md`.
- Append new entries to the existing file for that date.

## Rules

- Record meaningful milestones only: planning, implementation slices, reviews, commits, blockers, and recoveries.
- Keep entries concise and factual.
- Do not use the worklog as the authoritative next-step document; that belongs in `docs/11-handoff/ai-handoff.md`.
- When a session resumes after interruption, read the latest dated worklog file and the handoff together.

## Minimum Entry Content

- What changed
- Why it changed if not obvious
- Relevant artifacts or commit IDs when useful
